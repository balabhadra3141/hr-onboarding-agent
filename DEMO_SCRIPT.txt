[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
(pause)
Let me walk you through the system architecture.
Every user request starts at the START node and is immediately inspected by our Security Checkpoint. 
If the input contains prompt injections or is missing required consent, it routes directly to the Security Failed terminal node and is blocked.
Clean inputs route to the orchestrate_onboarding node, which invokes our lead hr_coordinator LLM agent.
The hr_coordinator uses AgentTools to delegate document-related requests to the document_specialist and training tasks to the training_specialist.
Both of these specialized sub-agents interact with our local database through our Model Context Protocol (MCP) server.
For routine informational queries, the coordinator routes directly to process_output.
But for critical actions requiring sign-off—like approving background checks—the workflow routes to the human_approval_node.
Here, the graph execution pauses and waits for manager input using a RequestInput interrupt, resuming only when the manager approves.
